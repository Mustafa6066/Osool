# Osool (أصول) — Design Context

## Design Context

### Users
Egyptian property investors and buyers — ranging from expat investors and domestic high-net-worth individuals to first-time buyers. They arrive with high-stakes financial decisions and need clarity, not clutter. Their context is often mobile-first, bilingual (English + Arabic with full RTL), and they're trying to compare developers, evaluate ROI, and make confident investment decisions in the Egyptian real estate market. The AI "CoInvestor" is their expert guide, not a chatbot novelty.

### Brand Personality
**Trustworthy. Modern. Expert.**

Osool is the calm, knowledgeable advisor in the room — never flashy, never gimmicky. It earns trust through data transparency, clear visual hierarchy, and an interface that feels as premium as the investments it facilitates. The tone is warm but professional — confident without being cold, approachable without being casual.

**Primary emotional goal: Confidence** — "I'm making the right investment decision."

### Aesthetic Direction
**Visual tone:** Premium minimalism inspired by Apple.com — liquid glass surfaces, generous whitespace, restrained animation with physics-based motion. Every element breathes.

**Visual signature:** The "liquid glass" frosted-panel system is the cornerstone aesthetic — translucent surfaces with subtle blur, soft borders, and inset highlights that give depth without heaviness.

**Reference:** Apple.com — specifically the premium material feel, confident typography, and the way complexity is hidden behind simplicity.

**Anti-reference:** Traditional real estate sites — cluttered listings, stock photos, generic card grids, overwhelming search filters, garish color schemes, and the "listing dump" pattern. Osool should feel more like a fintech product than a property portal.

**Theme:** Dark + Light mode (class-based toggle), with dark as the default. Both modes must feel equally polished.

### Color System
- **Primary:** Emerald (`#10B981`) — AI signature, trust, growth
- **Primary light:** `#34D399` — hover states, dark-mode primary
- **Primary dark:** `#059669` — pressed states, emphasis
- **Teal accent:** `#0D9488` — secondary AI accent, gradient endpoints
- **Deep teal:** `#267360` — brand depth
- **Chart tertiary:** Violet `#8B5CF6` / `#A78BFA` — sparingly, for data differentiation only
- **Semantic:** Success (emerald), Warning (amber `#F59E0B`), Danger (red `#EF4444`)
- **Surfaces:** Near-white `#FAFAFA` (light) / near-black `#09090B` (dark) with zinc gray spectrum

### Typography
- **Latin:** Inter (300–700) — clean, geometric, highly legible
- **Arabic:** Cairo (300–700) — designed specifically for Arabic UI text
- **Display:** Inter + Manrope stack for headings
- **Scale patterns:**
  - Hero: `text-4xl sm:text-5xl lg:text-6xl font-semibold tracking-tight leading-[1.05]`
  - Section labels: `text-sm font-semibold uppercase tracking-[0.18em]`
  - Body: `text-base sm:text-lg leading-relaxed`
  - Chat: `text-[14px] md:text-[15px]`
  - Micro: `text-[10px]` to `text-[12px]` for stats, badges, labels

### Spacing & Geometry
- **Grid:** 4pt base system (4, 8, 16, 24, 32, 48, 64px)
- **Border radii:** 8px (small inputs) → 12px (sub-elements) → 16px (cards) → 20–24px (panels, modals) → full (pills, badges, avatars)
- **Shadows:** Soft and layered — `0 1px 3px rgba(0,0,0,0.04)` at rest, `0 4px 16px` on hover, emerald glow `0 0 20px rgba(16,185,129,0.1)` for AI elements

### Motion
- **Library:** Framer Motion (primary) + CSS keyframes (ambient)
- **Spring physics:** `damping: 28, stiffness: 210–240` for interactive elements
- **Entry pattern:** `opacity: 0, y: 10–28` → `opacity: 1, y: 0` with staggered delays
- **Easing curves:** Expo out `cubic-bezier(0.16, 1, 0.3, 1)`, Quart out, Spring `cubic-bezier(0.34, 1.56, 0.64, 1)`
- **Ambient:** `float`, `breath`, `glass-shimmer`, `pulse-glow` for living-UI feel
- **Page transitions:** Spring-based with blur: `{ opacity: 0, y: 12, filter: 'blur(4px)' }`
- **Principle:** Motion communicates state and adds polish — never decorative excess

### Design Principles

1. **Clarity over density** — Every screen should breathe. Real estate data is complex; our job is to make it feel simple. Whitespace is a feature, not waste. If a view feels cluttered, it needs fewer elements, not smaller ones.

2. **Earned trust through transparency** — Show real data, real numbers, real comparisons. No vague marketing language. The UI should feel like it has nothing to hide — clean glass surfaces, visible data provenance, honest loading states.

3. **Premium materiality** — Every surface, shadow, and transition should feel considered. The liquid glass system, physics-based springs, and soft ambient animations create a tactile quality that says "this was built with care." Nothing should feel default or template-driven.

4. **AI as expert companion, not gimmick** — The CoInvestor AI should feel like a knowledgeable human advisor, not a chatbot. Its interface should be conversational but professional. Emerald accents mark AI presence subtly — a sparkle icon, a gentle glow — never overwhelming.

5. **Bilingual by design, not by afterthought** — Arabic RTL is not a mirror of the English layout. Typography, spacing, and flow should feel native in both directions. Cairo font for Arabic, proper `start`/`end` logical properties, tested in both languages.

### Accessibility
- **Standard:** WCAG AA minimum across all components
- **Contrast:** All text/background combinations must meet 4.5:1 (normal text) and 3:1 (large text) ratios
- **Focus:** Visible focus indicators on all interactive elements
- **Keyboard:** Full keyboard navigation support
- **Screen readers:** Semantic HTML, ARIA labels where needed, meaningful alt text
- **Motion:** Respect `prefers-reduced-motion` — disable non-essential animations
- **RTL:** Full bidirectional support with logical CSS properties
